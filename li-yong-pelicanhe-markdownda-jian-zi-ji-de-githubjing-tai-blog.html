利用Pelican和Markdown搭建自己的github静态Blog

2015-04-29

概述

从自己搭建Blog有一段时间了，其实为了方便我的Blog大部分都是从Github上荡下来的，有些地方没有著名版权，敬请谅解。

动态的Blog我也弄过一段时间，是用node.js开发的，当时也是由于没有找到合适的免费VPS（用过一段时间的DigitalOcean，也就没有继续下去了（对于学生来说还是有点贵的）。如果有需要，可以关注我的Github。

经过一段时间的摸索，完成了我的Github Page: http://blog.zzfan.info。

其实这个轩枫阁才是我一直想要搭建的Blog（很喜欢那种自己开发的可以不断自己的完善，而且有无限大的开发空间），但是我的方向不再网页开发，只能忍痛不去想那绚丽的让人眼花缭乱的Blog了

具体搭建

这里我就不很详细的列举出其中的细节了，直接上命令，是在Linux上搭建的。

- 安装软件

sudo apt-get install python python-dev python-pip
sudo apt-get install pelican Markdown

- 创建工作文件

mkdir pelican-blog
cd pelican-blog
pelican-quickstart

- 配置文件

经过上面的步骤，在blog文件夹里面会多出几个Pelican生成的文件，其中pelicanconf.py就是配置文件
为了方便，你可以把所有你需要操作的文件复制到这个文件夹里面，当作你Blog的操作文件夹。 首先你得在你的Github上创建一个Blog的repo，没有必要repo名字是"name.github.io",如果你非要把这个作为名字的话，我也不反对。然后如下：
cd blog
git clone http://github.com/zzfan/blog
cd blog
git chckout gh-pages

在这里需要说明的是在Github上创建repo的时候也不要使用Automatic Page Generator,因为没有这个必要，我们用不到（如果你想要用这个创建Blog的话其实很简单，具体的Github有给你很多模板）
