AF_INET和PF_INET的区别

Date 2016-04-06 Tags unp / 面试

摘要：刚写了一个原始套接字发送ipv6包的程序，平时一直用的 AF_INET协议簇，一直没有在意这些，感觉还是要总结下的。

在写网络程序的时候，建立TCP socket：
socket = socket(PF_INET, SOCK_STREAM, 0);

然后再绑定本地地址或连接远程地址时需要初始化sockaddr_in结构，其中指定address family时一般设置为AF_INET，即使用IP。
相关的头文件的定义： 

AF = Address family
PF = Protocol Family  
AF_INET = PF_INET

在Windows中的insock2.h中， 

#define AF_INET 0
#define PF_INET AF_INET

所以在windows中AF_INET与PF_INET完全一样。

而在Unix/Linux系统中，在不通过的版本中这两者有微小差别，对于BSD，是AF，对于POSIX是PF。
理论上建立socket时是指定协议，应该用PF_xxxx，设置地址时应该用AF_xxxx。当然AF_INET和PF_INET的值是相同的，混用也不会有太大的问题。也就是说你socket时候用PF_xxxx，设置的时候用AF_xxxx也是没关系的，这点随便找个TCP/IP例子就可以验证出来了。如下，不论是AF_INET还是PF_INET都是可行的，只不过这样子的话，有点不符合规范。 

//服务端开始建立socket描述符
if((sockfd = socket(PF_INET, SOCK_STREAM, 0)) == -1){
    fprintf(stderr, "Socket error: %s\n", strerror));
    exit(1);
}

//服务器端填充 sockaddr结构
bzero(&server_addr, sizeof(struct sockaddr_in));
server_addr.sin_family = AF_INET;
//server_addr.sin_family = PF_INET;
server_addr.sin_addr.s_addr = htonl(INADDR_ANY);
server_addr.sin_port = htons(portnumber);

在函数socketpair与socket的domain参数中有： 

AF_UNIX, AF_LOCAL, AF_INET, PF_UNIX, PF_LOCAL, PF_INET.

这几个参数有：AF_UNIX = AF_LOCAL, PF_UNIX = PF_LOCAL, AF_LOCAL = PF_LOCAL, AF_INET = PF_INET.
但是对于socketpair与socket的domain参数，使用PF_LOCAL系列，而在初始化套接口地址时，则使用AF_LOCAL.
例如：

z = socket(PF_LOCAL, SOCK_STREAM, 0); adr_unix.sin_family = AF_LOCAL;
