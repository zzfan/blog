Git的用法和遇到的问题

2015-05-02

Git的用法其实很简单，首先你得注册一个Github的账户，一般要在你的Github上创建一个repo。 那么git就有用武之地了，下面我们来介绍下具体的步骤以及注意的事项。

经验之谈

就我经常用到的git步骤如下：

git add .
git commit -m "你更改的描述"
git push

其实配置好了后就是这么简单，有时候在不同机器上会用到git pull、git add --all...,但是百度一下你就会了，而且git的提示功能是很Bug的一个存在，任何错误都会提醒你用什么命令，所以不用担心出现错误了。

有一点需要注意的是，有时候可能需要在git前面加sudo，这个你懂的。

具体步骤

初始配置

  • 配置git: 刚申请git的时候，必须按如下设置配置username和useremail

git config --global user.name "you name"
git config --global user.email you@yourdomain.example.com
git config --global push.default simple(可以有其他选项)

  • 安装ssh公钥: 用你在注册github时用的Email生成一个ssh公钥对

ssh-keygen -t rsa -C "your_email@example.com"

Enter passphrase (empty for no passphrase): 输入你在github上注册使用的密码
Enter same passphrase again: 再输入一次

然后会看到公钥和私钥分别被保存为id_rsa和id_rsa.pub，然后正是我们需要的。打开github网页，在账户设置(Account settings)中找到SSH keys项目，点击添加SSH key(Add SSH key)，把idrsa.pub中的内容复制到网页中的key里面。最后回到终端测试下： 

~$ ssh -T git@github.com

看到这个内容的时候，就说明成功了，git配置完毕:
The authenticity of host 'github.com (204.232.175.90)' can't be established.

RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.

Are you sure you want to continue connecting (yes/no)?

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

Git命令

  • git init
  • git add
  • git rm
  • git commit
    git commit -m ""
  • git status
    查看版本库的状态。
  • git log
    查看历史日志，包含每次版本变化。每次版本变化对应一个commit id。
  • git merge
    把服务器上下载来的代码和本地代码合并，或者进行分支合并：
    git merge repo/git-pages (是将服务器git库的git-pages分支合并到本地分支上)
  • git diff
    比较两个分支：git diff 分支1 分支2
  • git checkout
    切换分支： git checkout 分支名
  • git branch
    创建分支： git branch 分支名
    创建分支并且切换到创建的分支： git branch -b 分支名
    删除分支： git branch -D 分支名
    列出本地git库中的所有分支： git branch
